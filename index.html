ICS-Evo Email Verifier

The development team of ICS-Evo implemented a library to check if a string can be an email address or not. An email address satisfies all of the constraints below:

Description Valid Examples Invalid Examples
All characters belong to alphabet letters (lower and uppercase), numbers, ".", "@", "-", and "_". johndoe@uci.edu
12.34@uci.edu 		john?doe@uci.edu
There should be exactly one "@" (the part before "@" is called local-part and the part after that is a domain). johndoe@uci.edu john@doe@uci.edu
john.doe.uci.edu
The local-part and domain cannot be empty. j@uci.edu @uci.edu
johndoe@
There should be no two consecutive dots. johndoe@uci.edu john..doe@uci.edu
The domain name can be divided into several parts by ".". The last part consists of 2 to 4 alphabet letters. johndoe@uci.ed
johndoe@uci.educ 		johndoe@uci.e
johndoe@uci.education
All emails should be affiliated with UCI. In other words, the one before the last part of the domain should be "uci". Please note that the domain is case insensitive. johndoe@UCI.org
johndoe@ics.uci.com 		johndoe@gmail.com

BlackBox Testing

This assignment is about testing the ICS-Evo system. You are asked to test using a “black box” or specification-based testing approach. Although the code has been tested by the examples above, it seems there are some unraveled bugs. As a result, you are going to test this part of the code by providing a set of test cases. Each test case consists of two elements: the input and the expected output. In this case, the input is a string (for email address), and the output is a boolean value (the correctness of email). For each constraint, you have to provide at least two test cases with different expected outputs.

To test this functionality, you can use the tool below. Each row represents a test case, and you can add more test cases by clicking "Add Row." By clicking on "Run Tests," all test cases will be executed by the implemented code. If the expected output and the actual output of the system matches, then the corresponding row will become green, otherwise it becomes red.

Once you finished testing, click on " Test Cases" to download your test suite and don't forget to upload it on Canvas (HW4).
